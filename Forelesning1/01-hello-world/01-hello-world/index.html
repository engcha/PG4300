Ruby

A dynamic, open source programming language with a focus on simplicity and productivity. It has an elegant syntax that is natural to read and easy to write.
Ruby Official Website

Tolket

Ruby er et såkalt «tolket» (eng.: interpreted) språk. Dette betyr at det ikke kompileres – slik som f.eks. Java og C++ – men heller tolkes av kjøretidsmiljøet underveis, mens koden leses inn, som JavaScript og Python*.

Dynamisk typet

Ruby er «dynamisk typet», hvilket betyr at utvikleren ikke spesifiserer typene (f.eks. String foo;) variabler holder på som i statisk typede språk. 

i = 50 ; i.class
Fixnum 
i = "Foo" ; i.class
String

Kjøretidsmiljøet holder selv på type-hierarkiet, og "finner" selv ut hvilken klasse et objekt tilhører når koden kjøres.

Selv om dette ofrer litt av valideringen og typesikkerhet ved kompilering, så åpner det for mye høynivå-funksjonalitet som metaprogrammering, closures, ducktyping, m.m. i språket som vi kommer tilbake til senere.

Sterkt typet

Ruby er «sterkt typet», (strongly typed) hvilket betyr at i kjøretidsmiljøet ikke tillater brukere å anvende typer feil. Om du forsøker å mikse sammen ulike typer (f.eks. String og Fixnum) så vil ikke Ruby tillate dette, eller forsøke å finne ut av det, men heller kaste en exception.

Eksempelvis er JavaScript et svakt typet (weakly typed) programmeringsspråk. [repl.it]. Java (Kjøretidsmiljø) og C er andre eksempler.

Dynamisk typet, Sterkt typet

Et dynamisk typet språk er et språk hvor typen av en variabel kan endres (eller overskrives) på ethvert tidspunkt, også av kjøretidsmiljøet.

Et statisk typet språk er det motsatte. Du kan kun spesifisere typen ved kompilering.

Et sterkt typet språk er et språk som er strengt om hva du kan gjøre med dine typede variabler. Kjøretidsmiljøet vet hva typen er og hindrer en rekke feilsituasjoner*.

Et svakt typet språk det motsatte. Du kan mikse typer. Kjøretidsmiljøet vet ikke hva typen er.

Alt er objekter

I Java finnes primitive typer, slike som int, double og boolean. I Ruby er alle typer objekter av klasser.

Klasser er åpne

Dersom det skulle være behov for det, er det i Ruby mulig å åpne eksisterende klasser og legge til eller endre funksjonalitet.

Rails

Ruby on Rails® is an open-source web framework that's optimized for programmer happiness and sustainable productivity. It lets you write beautiful code by favoring convention over configuration.
Ruby on Rails Official Website

Skrevet i Ruby

Rails er skrevet i Ruby. Det er ikke et eget språk, men et rammeverk ment for å bygge web-applikasjoner. Rails fokuserer på produktivitet, og sikter på å gjøre utviklingen så smertefri som mulig for utvikleren.

Konvensjon fremfor konfigurasjon

Et av prinsippene bak Rails er det som heter convention over configuration. Det finnes allerede finnes mange standarder i Rails som bygger på eksisterende konvensjoner. Utvikleren skal slippe å konfigurere seg frem til disse.

Don't Repeat Yourself

Veldig mye av «magien» i Rails sørger for at utvikleren slipper å gjenta seg selv over flere steder. Et eksempel på dette er hvordan Rails – ved å se på kolonner i databasen – legger til ekstra metoder i klasser automatisk.

Observasjoner

Det er noen ting som er verdt å merke seg med Hello, World-eksempelet.

Vi trengte ikke å

  • kompilere koden
  • definere noen pakke eller namespace
  • lage noen klasser
  • opprette en main-metode
  • bruke parenteser når vi kalte på metoden puts

Ruby vs. Java

Syntaktiske forskjeller

  • Det finnes ingen primitive verdier.
  • Vi slipper å skrive semikolon på slutten av linjen.
  • Det er unødvendig med parenteser i metodekall.
    • Unntaket er ved tvetydig nøsting.
  • null heter nil.
  • i++ og i-- finnes ikke. Bruk heller i += 1 og i -= 1.
  • false og nil er de eneste verdiene som er «falsy.»
    • Alle andre verdier, også 0 er «truthy.»

unless

Det er mulig å bruke if !=, slik som vi er vant til fra f.eks. Java. Det ser imidlertid ofte mer lekkert ut med unless.

  # ...

Både if og unless kan også brukes til høyre for uttrykk:

puts "Logg inn" unless logged_in?

Lekkert!

case..when

Ligner på, men er mye kraftigere enn switch..case fra Java. NB! Ingen fall-through.

input = 2000
case input
when 1990...2000
  puts "Det skjedde på 90-tallet"
when Fixnum
  puts "Du skrev inn et heltall"
when "quit", "exit", "avslutt"
  exit
else
  puts "Ugyldig kommando: #{input}"
end

while fungerer som i Java, og until fungerer motsatt.

Installasjon

(Windows)

Git for Windows

Git for Windows gir deg et shell som ligner på Bash, som en finner i Linux og OS X. Det er mye kraftigere enn Windows' cmd, og tilgjengeliggjør også Git.

Last ned og installer Git for Windows og følg instruksjonene.

RubyInstaller

Den enkleste og tryggeste måten å få Ruby installert til Windows er å bruke RubyInstaller.

Last ned og installer RubyInstaller og følg instruksjonene.

Rails

Ruby on Rails installeres som en såkalt «gem» på følgende måte:

  1. Åpne Git Bash, som følger med Git for Windows.
  2. Kjør kommandoen gem install rails.
  3. Vent til installasjonen er ferdig.

Installasjon

(Linux)

Git

Installer Git fra din package manager (f.eks. apt-get eller yum).

Ruby Version Manager

RVM (Ruby Version Manager) gjør det enkelt å installere og håndtere flere Ruby-installasjoner på samme maskin.

  1. Sørg for at alle avhengigheter er installert.
  2. Åpne et terminalvindu.
  3. Installer RVM med denne kommandoen: 
    curl -L https://get.rvm.io | bash -s stable
  4. Følg instruksjoner som blir gitt etter installasjon.

Ruby / Rails

  1. Finn ut hvilke avhengigheter som trengs for din maskin med kommandoen rvm requirements.
  2. Følg instruksjonene som vises for å installere avhengighetene.
  3. Installer og sett opp Ruby med kommandoene under: 
    rvm install ruby-2.2.0
gem update --system
rvm use 2.2.0@rails --create
rvm --default 2.2.0
  4. Installer Rails med kommandoen 
    gem install rails

Installasjonsproblemer?

Dersom du har problemer med installasjon på Linux, er det sannsynligvis grunnet mangel på nødvendige pakker. Sjekk dokumentasjonen til RVM for instruksjoner.

Egentrening

Oppsett av miljø

Før du begynner på oppgavene bør du sette opp utviklingsmiljøet på maskinen din. Følg instruksjonene for Windows, Linux eller Mac OS X.

Try Ruby

Try Ruby er en nettside der du gjennomgår flere grunnleggende oppgaver om Ruby. Du skriver inn Ruby-kode inn i et interaktivt shell i nettleseren, så forteller serveren deg om du har skrevet riktig eller galt. Denne er veldig grunnleggende og anbefales å begynne med.

Ruby Koans

Ruby Koans er et sett med treningsoppgaver ment for å lære seg Ruby. De baserer seg på at det skrevet tester fra før som feiler (går rødt). Din oppgave er å gjøre så testene ikke lenger feiler (går grønt).

Når alle testene er grønne, er du ferdig. Dette er ikke veldig ulikt hvordan en bruker TDD – Test Driven Development – til daglig, og er en god praksis å venne seg til når en programmerer Ruby.

Last ned Ruby Koans eller gjør oppgavene online.
